# HadleyRead — Teaching Engine Specification v2

This document defines how HadleyRead works as a teaching system. Every decision in the code traces back to here. The AI coach operates inside this engine — it never overrides it.

---

## Core Architecture Principle

**The engine controls the structure. The AI controls the language.**

This principle is absolute and must be enforced in code, not just in prompts.

The AI coach must never control:
- Word choice
- Phase progression
- Baseline sequence
- Whether the user moves forward
- Whether a word is saved
- Whether a phase is complete
- Mastery levels
- Session structure

The AI coach may only generate short spoken responses within the current mode and current word/pattern chosen by the engine.

HadleyRead is not a chatbot. It is a structured adult reading tutor with an AI voice layer.

---

## Two Phases

### Phase 1 — Baseline Assessment

Runs once only. On the user's very first session.

**Purpose:** Find where the user is starting from. Not a test — a calibration.

**Opening message (hardcoded — not generated by GPT-4o):**
"Hi [name], welcome to HadleyRead. I'm going to show you a few words and I want you to say them out loud. This is just so I know where to start with you."

This string is constructed by the server with the user's name substituted in. It is passed directly to TTS. GPT-4o is not called for this message.

**GPT-4o is not used for coaching during baseline.**

During baseline, GPT-4o:
- Does not generate the opening message
- Does not provide coaching feedback between words
- Does not generate any speech at all
- Does not choose words
- Does not decide when baseline ends

The server controls the entire baseline sequence.

The only permitted use of GPT-4o during baseline is as a silent judgement fallback when the transcript is ambiguous. It may return structured assessment JSON only. It must not produce speech.

**Why silence matters:**
The opening instruction tells the user what to do. After that, silence between words protects the assessment and avoids making the user feel judged. The user hears one instruction, then works through the words. Nothing more.

---

**Baseline flow — step by step:**

1. Server shows baseline word silently on screen
2. User holds mic and attempts the word
3. Audio is transcribed by Whisper
4. Server judges transcript against target word using string matching
5. If judgement is uncertain, GPT-4o may be asked silently to classify the attempt — no speech generated
6. Outcome is stored in the attempts table
7. Next word appears immediately — no pause, no feedback
8. Repeat until stopping rule is met
9. No teaching begins until baseline is fully complete and all data is saved

---

**Baseline word sequence — adaptive:**

Start with these 5 words in order:

| # | Word | Level | Pattern |
|---|------|-------|---------|
| 1 | cat | 1 | Short vowel CVC |
| 2 | stop | 2 | Consonant blend |
| 3 | shop | 3 | Digraph |
| 4 | gate | 4 | Magic E |
| 5 | rain | 5 | Vowel pair |

If the user gets all 5 correct, continue:

| # | Word | Level | Pattern |
|---|------|-------|---------|
| 6 | card | 6 | R-controlled vowel |
| 7 | working | 7 | Common ending |
| 8 | induction | 8 | Multi-syllable |
| 9 | confirmed | 9 | High value adult word |

**Stopping rules:**
- Stop when the user gets 2 wrong in a row
- Stop when all 9 words are exhausted
- Maximum 9 words — never exceed this

---

**Baseline judgement JSON — used only when transcript is ambiguous:**

```json
{
  "targetWord": "cat",
  "heardAs": "cap",
  "outcome": "correct | close | wrong | unclear",
  "confidence": "high | medium | low",
  "likelyError": "initial_sound | vowel_sound | final_sound | added_sound | omitted_sound | whole_word | unclear | none"
}
```

Rules:
- If confidence is low, do not mark the word wrong automatically
- If outcome is unclear, present another word at the same level rather than penalising the user
- GPT-4o must not generate speech during baseline judgement
- The judgement result is for the engine only — never shown to the user

---

**On baseline completion:**

- If errors begin at a level, set current_phase and current_pattern to the earliest unstable level/pattern
- If the user gets all 9 correct, set current_phase to Phase 8 or Phase 9 review — not beginner content
- If results are unclear, start slightly below the uncertain level rather than above it
- Record all baseline words in the attempts table with outcome and confidence
- baseline_done must only be set to 1 after all baseline data has been successfully written to the database

**Hard rule:**
Never begin Phase 2 teaching unless baseline_done = 1 and both current_phase and current_pattern are present in the database.

---

**If baseline is interrupted:**
- Store baseline_progress in the database — which words attempted, which outcomes
- On next login: "Welcome back [name]. Let's carry on from where we were." — resume from next unattempted word
- Never restart the baseline from scratch once any words have been attempted

---

### Phase 2 — Teaching

Begins immediately after baseline completes, or from the second session onward.

The engine determines what happens in every session. The AI operates within it.

---

## Session State

Every active session tracks the following:

```json
{
  "currentMode": "baseline | rule_intro | practice | review | close",
  "currentPhase": "phase_1_short_vowels",
  "currentPattern": "short_a",
  "ruleIntroducedThisSession": true,
  "currentContext": "isolated | sentence | wild"
}
```

**Mode rules:**
- `baseline` — first session only, server-controlled entirely
- `rule_intro` — tutor introduces the current pattern once, then transitions to practice
- `practice` — words appear silently, user attempts, AI gives feedback
- `review` — mixed words from previous phases, same practice mechanic
- `close` — session end, tutor says goodbye, progress saved

**Rule introduction:**
- Each new pattern gets one short rule introduction before practice begins
- The rule is not repeated before every word
- If the user struggles across multiple words, the engine may trigger a re-teach — this is a deliberate event, not automatic repetition
- The tutor teaches the rule, not the word
- Practice words always appear silently

## Mastery Levels

Every word has a mastery level stored in the words table.

| Level | Colour | Label | Definition |
|-------|--------|-------|------------|
| 0 | Red | Introduced | Seen but not yet correct in any context |
| 1 | Orange | Emerging | Correct in isolated context 2+ times, across 2+ sessions |
| 2 | Green | Secure | Correct in isolated and sentence context, across 3+ sessions |
| 3 | Dark green | Mastered | Correct in all three contexts, across 4+ sessions, with at least one session 7+ days after introduction |

**The three contexts:**

1. **Isolated** — word appears alone on screen

   Example: `stop`

2. **Sentence** — target word inside a simple real sentence

   Example: `Stop here.`

3. **Wild** — word in a realistic adult context: text message, sign, form, notice, app screen, label

   Examples:
   - Text: "Can you stop at the shop?"
   - Sign: STOP / Site entrance
   - Form: Previous address: ___
   - Label: Best before: 12 June

**Context rules:**
- Sentence and wild contexts are only introduced once the word has enough isolated success
- Wild context must never become a comprehension task
- The user is practising reading, not explaining meaning
- The tutor must never ask "What does it mean?"

A word only moves up a mastery level when the conditions are met across multiple sessions — not within a single session.

---

## Progression Rules

### Moving forward to the next phase
All three conditions must be true:
- 80% of words in current phase are at green or above
- No word in current phase is still at red after 3 sessions
- At least one session has passed since the phase was introduced

### Staying in the current phase
Either condition triggers:
- More than 20% of words still at red or orange
- User has failed the same word 3 times in a row

### Going back a phase
Triggered when:
- Words from a previous phase appear wrong in sentence or wild context
- Drop affected words back one mastery level and schedule focused work

---

## Stuck Detection

### After 2 failures on the same word in one session:
- Save it — status = 'saved', outcome = 'saved'
- Move to next word immediately
- Do not return to it in the same session

### After a word has been saved 3 times across different sessions:
- Flag as a problem word — status = 'problem'
- Break it down — identify component sounds causing difficulty
- Address those components in simpler words first
- Only return to the problem word once its components are secure

### If a whole pattern is difficult after 2 sessions:
- Step back to simpler words within the same pattern
- Introduce easier examples before returning to harder ones

### Hard limits — never exceeded:
- Never present the same word more than twice in one session
- Never present more than 3 saved words in one session
- Always mix in words the user knows confidently — sessions must never feel like they are going backwards

---

## Mixing Phases

### When mixing begins — all three conditions must be true:
- User has at least one phase at 80% green or above
- User is at least into Phase 3
- At least 3 sessions completed

### Session composition when mixing:
- 60% current phase words
- 20% review words from previous phases (green but not dark green — prioritised)
- 20% mastered words in sentence or wild context

### Rules for review words:
- Prioritise green words from previous phases
- Include orange words from previous phases if any exist
- Never include red words from previous phases in a mixed session

### How it feels:
- The tutor never announces that mixing is happening
- It feels like normal practice
- If a previously secure word comes back wrong, drop it to orange and schedule focused work

---

## Session Structure

### Session length:
- No fixed time limit
- Engine aims for a natural stopping point every 15–20 words
- After 20 words the tutor offers a natural close — never forces it

### How a session opens:
- Start with 2–3 words the user knows well
- One or two sentences from the tutor about what they are working on today
- Then straight into the first word

### Word ordering within a session:
1. 2–3 known words (mastery level 2 or 3) — confidence builder
2. Current phase words — the main work
3. Review words mixed in
4. End with 1–2 known words — close on solid ground

### How a session closes:
- After the final word the tutor says something brief and calm
- Example: "That's it for today. Same time tomorrow."
- Never: praise, scores, or encouragement to keep going
- Progress summary shown on screen immediately after

### Session frequency tracking:
- 7+ days away: "It's been a while. Let's pick up where we left off." — warm up with known words first
- 30+ days away: treat as near-returning, warm up across multiple phases before new material

## AI Coach — Structured Output

### Teaching mode — every GPT-4o response must return valid JSON:

```json
{
  "speech": "maximum 2 sentences, under 40 words",
  "wordOutcome": "correct | close | wrong | saved | null",
  "targetWord": "the word just attempted, or null",
  "nextAction": "retry_same_word | next_word | save_word | teach_rule | start_review | end_session"
}
```

**wordOutcome definitions:**
- `correct` — user read the target word accurately enough to count as correct
- `close` — near the word but made a small sound error
- `wrong` — did not decode the word accurately
- `saved` — word has hit the stuck rule and is being saved for later
- `null` — tutor is not responding to a word attempt (rule intro, session open/close)

**nextAction rules:**
- `correct` usually leads to `next_word`
- `close` usually leads to `retry_same_word`
- `wrong` usually leads to `retry_same_word` unless stuck rule has been reached
- `saved` always leads to `save_word`
- `null` may lead to `teach_rule`, `next_word`, `start_review` or `end_session`

**The engine validates nextAction before acting. The engine may override nextAction if it conflicts with hard rules. The engine is the final authority on progression.**

## AI Coach — Server Validation

Applied to the `speech` field only before it goes to TTS. Internal JSON fields, system context, and prompts are not subject to this validation.

### Speech length:
- Under 40 words
- 2 sentences or fewer

### Forbidden learner-facing words:
well done, great job, excellent, amazing, brilliant, fantastic, good job, proud, clever, superb, perfect, awesome, phoneme, grapheme, digraph, blend, test, lesson, school, teacher, children, class, kids, pupil, student

### Speech must start with:
- `correct` → "Yep."
- `close` → "Almost."
- `wrong` → "Let's slow it down."
- `saved` → "I've saved that one."
- Second attempt correct → "There it is."
- `null` → no constraint on opening word

### Target word must not be modelled aloud:
Reject speech if it contains any of these patterns:
- "The word is [targetWord]"
- "Say [targetWord]"
- "Try [targetWord]"
- "It says [targetWord]"
- "That word is [targetWord]"
- "That was [targetWord]"
- "You read [targetWord]"
- "[targetWord]" as a standalone modelled utterance

**Exception:** Do not automatically reject if the target word is a very common short word (a, in, on, at, it, is, to, the). The validation prevents the tutor from modelling the answer — not from using common words in natural speech.

### Rejection handling:
- Maximum 2 rejections before falling back to hardcoded response
- Log every rejection with the reason — used to improve the prompt over time

### Hardcoded fallbacks:

| Situation | Fallback |
|-----------|----------|
| Correct | "Yep. That pattern works every time." |
| Close | "Almost. Try it again." |
| Wrong | "Let's slow it down. Try the first sound, then the rest." |
| Second attempt correct | "There it is." |
| Saved | "I've saved that one. We'll come back to it." |
| Rule introduction | "Here's how this one works. [pattern rule in plain English]" |
| Session close | "That's it for today. Same time tomorrow." |

All fallbacks must:
- Pass the forbidden word check
- Not contain the target word as a modelled utterance
- Stay under 40 words

## Phonics Phases — Teaching Sequence

The engine uses this as a map, not a script. It adapts based on evidence but never breaks the teaching rules.

### Phase 1 — Short vowels
Rule: vowel between two consonants makes a short sound.

| Pattern | Practice words |
|---------|----------------|
| Short A | bag, van, tap, mat, cap, ran, flat, clap, trap, man, fan, rat, hat, jam, lap |
| Short I | bin, hit, sit, tip, grip, slip, trim, grit, skip, drip, pin, win, bit, fit, lip |
| Short O | job, top, mop, rod, log, pot, dot, cot, hot, lot, box, fox, pop, hop, rob |
| Short U | cup, run, cut, mud, bus, bug, jug, mug, rug, tug, bun, fun, gun, sun, but |
| Short E | bed, set, pen, leg, step, help, melt, belt, shelf, get, let, net, wet, fed |

### Phase 2 — Consonant blends
Rule: two consonants together — say both sounds, then the rest.

| Pattern | Practice words |
|---------|----------------|
| S blends | stop, step, slip, skill, span, strap, split, spray, skin, spin, spit, spot |
| L blends | flat, plan, clip, block, plug, flag, glad, clamp, flip, flop, plod, clap |
| R blends | grip, trim, drop, crop, press, brass, graft, drip, drag, drum, frog, grab |
| End blends | belt, hand, lift, melt, lamp, damp, shift, trust, best, fast, last, dust |

### Phase 3 — Digraphs
Rule: two letters make one sound.

| Pattern | Practice words |
|---------|----------------|
| SH | shop, shift, shelf, wish, rush, crush, brush, flash, dish, fish, shack, shin |
| TH | this, that, with, then, path, thick, both, cloth, bath, moth, thud, thin |
| CH | chat, check, chip, much, such, fetch, bench, chest, chop, chin, rich, march |
| CK | back, lock, kick, pack, rack, dock, sock, tick, pick, buck, duck, neck |

### Phase 4 — Magic E
Rule: silent E at end makes the vowel say its name.

| Pattern | Practice words |
|---------|----------------|
| A-E | name, gate, safe, came, late, plate, frame, trade, brave, wave, cave, rake |
| I-E | time, site, fine, ride, wide, pride, drive, slide, hide, tide, tile, mile |
| O-E | home, note, code, role, pole, smoke, stone, drove, bone, tone, bore, core |
| U-E | tune, rule, cube, fuse, crude, prune, flute, cute, rude, duke, plume, tube |

### Phase 5 — Vowel pairs
Rule: two vowels together make one long sound.

| Pattern | Practice words |
|---------|----------------|
| AI/AY | rain, play, stay, train, pain, main, claim, drain, spray, brain, chain, snail |
| EA/EE | read, keep, lead, team, clean, mean, sleep, steel, feel, deal, heal, cream |
| OA/OW | road, flow, grow, load, coat, goal, groan, show, blow, boat, float, coast |
| OO | tool, roof, food, proof, mood, cool, pool, smooth, tooth, moon, spoon, room |

### Phase 6 — R-controlled vowels
Rule: R changes the vowel sound before it.

| Pattern | Practice words |
|---------|----------------|
| AR | car, hard, farm, bar, card, yard, harm, sharp, start, dark, mark, park |
| OR | for, sort, short, more, work, port, form, storm, force, score, torn, born |
| ER/IR/UR | her, first, burn, term, girl, turn, hurt, curl, shirt, fern, firm, fur |

### Phase 7 — Common endings
Rule: recognise and read common word endings.

| Pattern | Practice words |
|---------|----------------|
| -ING | working, coming, running, lifting, checking, building, driving, fixing, calling |
| -ED | worked, moved, lifted, checked, called, signed, fixed, passed, asked, helped |
| -ER | worker, driver, builder, loader, faster, harder, stronger, longer, cleaner |
| -LY | safely, clearly, firmly, daily, weekly, roughly, badly, fully, quickly, slowly |

### Phase 8 — Multi-syllable words
Rule: break into parts, say each part, put together.

| Level | Practice words |
|-------|----------------|
| Two syllable | forklift, scaffold, contract, payment, notice, permit, helmet, carpet, garden, market |
| Three syllable | induction, instruction, protection, inspection, tomorrow, remember, together, hospital |
| Four syllable | communication, organisation, qualification, authorisation, identification |

### Phase 9 — High value adult words
Real words from everyday adult life. Taught in real context.

Full word list in hadleyread-tutor-spec.md.

## First Build Slice

The first working version must include only the following. Nothing else.

**Include:**
- Register and login
- Baseline assessment — adaptive 5–9 word sequence
- Baseline interruption and resume
- Storage of all baseline results in database
- Automatic transition into teaching on baseline completion
- One teaching pattern only, chosen from baseline result
- Rule introduction spoken once per pattern per session
- 5–8 isolated practice words shown silently
- User reads each word aloud via mic
- AI gives validated short feedback within hard rules
- Words marked correct, close, wrong or saved
- Session ends at a natural stopping point
- Basic progress saved to database

**Do not build yet:**
- Full progress calendar
- All nine phases
- Mixed phase sessions
- Sentence context
- Wild context
- Animations
- B2B dashboards

This keeps the first testable version focused and prevents overbuilding.

---

## What This System Is Not

- Not a game — no stars, badges, streaks, levels or scores
- Not a test — the user is never shown a word without having been taught the pattern first, except during baseline
- Not a children's app — no reference to school, teachers, reading ages or Jolly Phonics
- Not a comprehension tool — never ask if the user understood something
- Not a fixed path — the engine adapts based on evidence, but it never breaks the teaching rules
